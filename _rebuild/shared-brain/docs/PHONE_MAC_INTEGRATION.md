# 手机+Mac 协同方案设计

> 目标：让 iPhone + Mac + WorkBuddy + OpenCode 形成 **1+2 >> 3** 的协同效应

---

## 核心愿景

**你在任何地方（手机/电脑），都能无缝继续工作**：
- 在地铁上用手机看微信/视频号 → 发现投资机会 → 一键发送到 Mac 上的 WorkBuddy/OpenCode
- 在 Mac 上 WorkBuddy 完成分析 → 自动推送到手机微信 → 随时查看
- 手机拍摄的白板/文档 → 自动同步到 Mac 工作区 → WorkBuddy/OpenCode 直接处理

---

## 方案设计（三层架构）

### 第一层：实时通知层（已部分实现）

**目标**：Mac 上的事件实时推送到手机

| 事件 | 推送方式 | 状态 |
|:-----|:---------|:-----|
| 内容管线阻塞 | 企微通知 | ⏳ 需配置 wecom 连接器 |
| 新任务完成 | 企微通知 | ⏳ 需配置 wecom 连接器 |
| 研究报告完成 | 微信推送 | ⏳ 需配置公众号 API |
| 图表生成完成 | iMessage/企微 | ⏳ 待实现 |

**实现方式**：
1. **企微通知**（推荐）：WorkBuddy 自动化 → 企微 API → 手机企微 APP
2. **微信推送**：WorkBuddy → 公众号草稿箱 → 手机微信查看
3. **iMessage**：Mac 上运行 AppleScript → 发送 iMessage 到 iPhone

---

### 第二层：快速输入层（手机 → Mac）

**目标**：手机上的灵感/信息快速传递到 Mac 工作区

#### 方案 A：微信文件传输助手（最简单）

**工作流**：
1. 手机微信 → 文件传输助手 → 发送文字/图片/文件
2. Mac 微信 → 自动下载到指定目录
3. 文件监听器检测到新文件 → 触发 WorkBuddy/OpenCode 处理

**优点**：
- ✅ 无需额外 APP
- ✅ 微信本身就是你常用的工具
- ✅ 支持文字、图片、文件、链接

**缺点**：
- ⚠️ 需要 Mac 微信保持登录
- ⚠️ 文件传输助手容易被其他消息淹没

#### 方案 B：WorkBuddy 小程序（已可用）

**工作流**：
1. 手机微信 → WorkBuddy 小程序
2. 输入任务/上传文件
3. 小程序调用 WorkBuddy API → Mac 上的 WorkBuddy 接收任务

**优点**：
- ✅ 专为 WorkBuddy 设计
- ✅ 可以查看任务状态
- ✅ 支持文件上传

**缺点**：
- ⚠️ 需要配置 WorkBuddy API（可能需要内网穿透）

#### 方案 C：iCloud 共享文件夹（Apple 生态）

**工作流**：
1. iPhone 保存文件到 iCloud Drive/`workbuddy-inbox/`
2. Mac 自动同步
3. 文件监听器检测到新文件 → 触发处理

**优点**：
- ✅ Apple 原生支持，无需额外配置
- ✅ 自动同步，无需手动操作
- ✅ 支持所有文件类型

**缺点**：
- ⚠️ 需要 iPhone 和 Mac 都登录同一 iCloud 账号
- ⚠️ 只能传文件，不能传文字指令

#### 方案 D：快捷指令（Quick Actions）

**工作流**：
1. iPhone 分享菜单 → 自定义快捷指令
2. 快捷指令将内容上传到 Mac（通过 iCloud/API）
3. Mac 上的 WorkBuddy/OpenCode 接收并处理

**优点**：
- ✅ 系统级集成，任何 APP 都能调用
- ✅ 支持文字、图片、网页、文件
- ✅ 可以自定义处理逻辑

**缺点**：
- ⚠️ 需要编写快捷指令（一次性工作）
- ⚠️ 需要配置接收端（Mac 上的监听服务）

---

### 第三层：无缝切换层（上下文延续）

**目标**：在手机和 Mac 之间无缝切换，上下文不丢失

#### 方案 A：统一会话日志（已实现）

**工作流**：
1. OpenCode/WorkBuddy 每次操作都写入 `brain/context-log.json`
2. 你在手机上查看 → 读取最新的日志条目
3. 切换回 Mac → 读取日志 → 恢复上下文

**优点**：
- ✅ 已实现，无需额外开发
- ✅ 完整的操作历史

**缺点**：
- ⚠️ 需要在手机上能访问 `brain/context-log.json`（通过 iCloud/小程序）

#### 方案 B：企微/微信 每日简报

**工作流**：
1. WorkBuddy 每天定时生成工作简报
2. 推送到企微/微信
3. 你在手机上查看简报 → 了解当天工作进展

**优点**：
- ✅ 主动推送，无需手动查看
- ✅ 适合移动场景

**缺点**：
- ⚠️ 需要配置推送连接器
