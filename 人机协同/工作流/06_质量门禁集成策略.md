# 质量门禁集成策略

> `qc-gate.mjs` 的统一集成方式

---

## 当前状态

- `scripts/qc-gate.mjs` ✅ 已存在（支持 article/video/ppt/astro/bundle 类型）
- 但未被 `content-pipe.mjs` 自动调用
- 也未被三件套生产流程调用

## 集成方式

在每个产出文件生成后，立即调用：
```bash
node scripts/qc-gate.mjs <产出文件路径> --json
```

### 集成点

| 产出阶段 | 文件 | 调用时机 |
|:---------|:-----|:---------|
| 公众号文章 | `*_公众号文章.md` | 推送草稿箱前 |
| 网站文章 | `*_网站文章.md` | md-to-astro 转换前 |
| .astro | `*.astro` | articles.ts 注册前 |
| PPT | `*.pptx` | 生成完毕后 |
| Bundle | `*.bundle.json` | 注册前 |

### 失败处理

```
QC 失败 → 打印警告（非阻断）
QC 严重失败（文件为空/格式错误） → 标记 pipeline.blocked=true
```
