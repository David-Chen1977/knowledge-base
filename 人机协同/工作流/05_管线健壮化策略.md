# 管线健壮化策略

> 解决"三件套"生产中 deep agent 超时、管线中断等问题

---

## 问题诊断

| 问题 | 场景 | 根因 |
|:-----|:-----|:-----|
| deep agent 超时 | 异步生成网站文章 | background task 30分钟 idle timeout，且无重试 |
| 管线依赖链断裂 | 公众号→网站→PPT 串行 | 任一环节卡住则后续全部阻塞 |
| 信息不一致 | 网站文章与公众号数据不同 | 两个 agent 独立生成，未共享结构化数据 |
| 降级模糊 | agent 失败时无明确 fallback | 没有"最低可交付"标准 |

## 两阶段拆分策略

### 阶段 A（同步 · 必须完成）

```
公众号文章 → 封面图上传 → 草稿箱推送
```
- 始终在主会话中直接执行，不委托给 background agent
- 通常 5 分钟可完成
- **如果此阶段失败** → 整批阻塞（call for help）

### 阶段 B（异步 · 可降级）

```
网站文章 → md-to-astro → 网站部署
PPT生成
```
- 委托给 `task(category="deep", run_in_background=true)` 并行执行
- 每个 agent 独立，互不依赖
- **如果此阶段失败** → 标记 pipeline.blocked=true，不阻塞阶段 A

## 超时与重试策略

```javascript
// 伪代码 — 异步 agent 调用时应遵循的模式
function safeSpawn(agentPrompt, timeoutMs = 300000) {
  try {
    const result = await task({
      category: "deep",
      run_in_background: true,
      prompt: agentPrompt,
      // 注意: background task 的 idle timeout 为 30分钟
      // 策略：如果超时，标记降级，不卡住管线
    });
    return { ok: true, result };
  } catch (err) {
    return { ok: false, error: err.message, degraded: true };
  }
}

// 使用方式
const websiteResult = await safeSpawn("生成网站文章...");
if (!websiteResult.ok) {
  // 降级：手动补写 or 跳过
  await syncState("pipeline", "blocked", "网站文章生成超时，等待人工补写");
}
```

## 降级矩阵

| 场景 | 降级行为 | 影响 |
|:-----|:---------|:-----|
| 网站文章 agent 超时 | 标记 blocked，人工补写 | 网站部署延迟 |
| PPT agent 超时 | 跳过，标记"需手动生成" | 无 PPT 交付 |
| 公众号图片上传失败 | 重试 2 次 → 跳过图片推送 | 需手动补上传 |
| 网站 build 失败 | 打印错误日志，不 push | 等待修复后重试 |
| SEC/Fred MCP 超时 | 跳过该数据源，用已有知识 | 数据可能不完整 |

## 最低可交付标准

```
Pipeline 必须 deliver:
  ✅ 公众号文章（草稿箱）— 核心产出，不可跳过
  
Pipeline 应该 deliver:
  ✅ 网站文章（已部署）— 高优先级
  ✅ PPT（可打开）— 中优先级

Pipeline 可选 deliver:
  ✅ 视频脚本/视频 — 低优先级
```

## 验证步骤

1. `lsp_diagnostics` 检查所有新文件无错误
2. 如项目有 build 命令 → exit code 0
3. 如有测试 → 通过（或注明预存失败）
