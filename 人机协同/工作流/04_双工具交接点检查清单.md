# 双工具交接点检查清单

> 当 OpenCode 完成一个工作段、需要 WorkBuddy 接续时，必须完成的检查项

---

## OpenCode → WorkBuddy 交接

```
□ 1. state.json 已更新
   - openCode.status = "idle"
   - pipeline.phase = 当前阶段（research/writing/publish）
   - operationLog 已追加最新操作

□ 2. 产出文件完整
   - 公众号文章: 已推送草稿箱（有 media_id）
   - 网站文章: 已 git push（Cloudflare 自动部署中）
   - PPT: 文件可打开（页数≥5）

□ 3. 知识库已更新
   - 新研究已写入 知识库/ 对应目录
   - 如有回流价值，更新 .回流记录.json

□ 4. 无残留阻塞
   - pipeline.blocked = false
   - 如需人工操作，已在 state.json 备注
```

## WorkBuddy → OpenCode 交接

```
□ 1. workBuddy.status = "idle"

□ 2. 可视化图表已生成至 artifacts/

□ 3. 归档完成（腾讯文档）

□ 4. 企微通知已发送（如适用）

□ 5. 如有新任务需求，已记录至 state.json 备注
```
