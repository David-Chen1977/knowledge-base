# 三件套生产工作流

> 适用于：将一份深度研究报告扩展为公众号文章 + 网站文章 + PPT

---

## 入口条件

- [ ] 研究报告已完稿（.md 格式，保存在 `三件套输出/`）
- [ ] 主题明确，有核心数据和观点
- [ ] 写作风格参考已确认（`知识库/写作风格参考-陈道雷.md`）

## 阶段一：公众号文章（同步 · OpenCode）

```
输入: 研究报告 .md
输出: 公众号文章 .md + 封面图 .jpg
时间: ~5 分钟
```

**步骤：**
1. 读取研究报告，提取核心观点和数据
2. 遵循公众号风格（口语化、短段落、开头钩子、互动结尾）
3. 生成封面图（900×500，深色背景+金色标题）
4. 调用 `wechat_publisher.py --cover cover.jpg` 推送草稿箱
5. 更新 state.json: `sync-state.mjs opencode idle "公众号文章完成"`

**验证：** 草稿箱可预览，媒体资源正常显示

## 阶段二：网站文章 + PPT（异步 · OpenCode）

```
输入: 研究报告 .md
输出: 网站文章 .md + .astro + PPT .pptx
时间: ~10 分钟（含异步等待）
```

**步骤：**
1. 生成网站文章（专业有温度、数据驱动、含表格/ASCII art）
2. 创建 bundle JSON → `md-to-astro.mjs` 生成 `.astro` 页面
3. 注册 `articles.ts`
4. `npm run build && git push` 部署
5. 同步生成 PPT（Python `gen_pptx_v2.py` 或 `gen_pptx_*.py`）
6. 更新 state.json

**注意：** 网站文章 + PPT 可并行生成（用两个后台 agent）

## 阶段三：归档与通知（WorkBuddy）

```
输入: state.json 中 pipeline.phase === "publish"
输出: 图表 + 腾讯文档 + 企微通知
```

**步骤：**
1. 读取 `context/state.json` 获取产出清单
2. 运行 `workbuddy-visualization-bridge.mjs` 生成图表
3. 同步产出到腾讯文档归档
4. 企微推送发布通知

## 阶段四：发布（人工）

- 公众号：去 `mp.weixin.qq.com` → 草稿箱 → 预览 → 发布
- 网站：Cloudflare Pages 自动部署（push 即生效）

## 质量门禁

每阶段完成后自动运行：
```
node scripts/qc-gate.mjs <产出文件>
```

| 产出类型 | 检查项 |
|:---------|:-------|
| 公众号文章 | 标题≤64字、无Markdown残留、封面已上传 |
| 网站文章 | 结构完整、链接有效 |
| PPT | 可打开、页数≥5、无空页 |
| .astro | 语法正确、articles.ts 已注册 |

## 失败恢复

| 失败场景 | 处理方式 |
|:---------|:---------|
| Agent 超时 | 标记 state.json pipeline.blocked=true → 人工介入重试 |
| 公众号 API 错误 | 检查 AppSecret 是否过期 → 更新 `.env` → 重试 |
| 网站 build 失败 | `npm run build` 查看错误 → 修复 → 重试 |
| PPT 生成失败 | 降级为手动 PowerPoint 生成（不影响发布） |
