# 双工具心跳同步规范

> 确保 OpenCode 和 WorkBuddy 之间始终知道对方"在做什么"

---

## state.json 状态机

```
opencode.status
  ├── idle       → 空闲，等待用户指令
  ├── working    → 正在执行任务
  └── blocked    → 阻塞（需人工或WorkBuddy介入）

workbuddy.status
  ├── idle       → 空闲
  └── working    → 正在处理

pipeline.phase
  ├── idle       → 无活跃管线
  ├── research   → 研究中
  ├── writing    → 写作中
  ├── review     → 审核中
  ├── publish    → 发布中
  └── blocked    → 阻塞
```

## OpenCode 侧规范

**每次完成重要操作后**，调用：
```bash
node /Users/Admin/.workbuddy/shared-workspace/scripts/sync-state.mjs \
  opencode idle \
  "任务简述" \
  "产出文件/URL"
```

**开始新任务前**：
```bash
node /Users/Admin/.workbuddy/shared-workspace/scripts/sync-state.mjs \
  opencode working \
  "冷却液材料公众号文章生成" \
  ""
```

**遇到阻塞时**：
```bash
node /Users/Admin/.workbuddy/shared-workspace/scripts/sync-state.mjs \
  opencode blocked \
  "公众号 API 凭证过期" \
  ""
```

## WorkBuddy 侧规范

**定时轮询**（每 5 分钟）：
1. 读取 `state.json`
2. 检查 `openCode.status` 和 `pipeline.phase`
3. 如有新产出 → 执行可视化/归档/通知
4. 更新 `workbuddy.status` 和 `workbuddy.lastHeartbeat`

## 自动触发规则

| state.json 状态 | 自动触发的动作 |
|:----------------|:--------------|
| pipeline.phase === "publish" | WorkBuddy 生成可视化图表 |
| pipeline.phase === "publish" | 归档到腾讯文档 |
| openCode.status === "blocked" | WorkBuddy 发送企微通知给用户 |
| pipeline.phase === "idle" > 30min | 空闲状态，无需操作 |
