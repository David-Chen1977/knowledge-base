# PPT 生成质量检查标准

> 每次用 python-pptx 生成 PPT 后，逐条检查以下项目。

---

## 一、布局与溢出

- [ ] 所有表格列宽总和 ≤ 表格总宽度
- [ ] 所有元素左边界 ≥ 0.5"，右边界 ≤ 12.8"（页面 13.333" 宽）
- [ ] 所有元素上边界 ≥ 0.2"，下边界 ≤ 7.3"（页面 7.5" 高）
- [ ] 相邻文字框垂直间距 ≥ 0.1"（防止重叠）
- [ ] 表格行数 × 每行高度 ≤ 表格总高度
- [ ] 一组元素（表格+下方文字+图标）总高度 ≤ 可用区域高度

## 二、颜色与字体

- [ ] 主色使用 `#C00000`（标题/强调）
- [ ] 正文使用 `#2C2C2C` 或 `#343A40`
- [ ] 辅助文字使用 `#8D99AE` 或 `#4A5565`
- [ ] 字体统一使用「微软雅黑」（仅英文数字可用 Franklin Gothic Medium）
- [ ] 表格表头使用深色背景（`#101828`）+ 白色文字

## 三、页眉元素

- [ ] 左侧装饰方块：白底黑框 + 右下角红色小方块突出
- [ ] 标题：红色 `#C00000` 26pt 粗体，y≈0.37"
- [ ] 红色分隔线：y≈0.99"，与标题左对齐
- [ ] 右上角汇竑 logo：右边缘与分隔线右边缘对齐（x=12.42"）
- [ ] 页码：右下角 10pt `#8D99AE`

## 四、主页视觉一致性

- [ ] 所有内容页使用统一白色背景（无深色页面）
- [ ] 所有内容页调用 ph() 生成标准页眉
- [ ] 表格交替行颜色：白/浅灰（`#F5F7FA`）
- [ ] 步骤卡片：浅灰底 + 红色顶条 + 编号圆（栗色 `#8B0000`）

## 五、内容与展示

- [ ] 正文文字大小 ≥ 10pt（确保可读性）
- [ ] 标题文字大小 ≥ 20pt
- [ ] 表格文字 ≤ 11pt
- [ ] 无多余或冲突的视觉元素（如同时使用序号 emoji 和装饰方块）
- [ ] 文字不超出文本框边界（word_wrap=True）

## 六、检查清单使用方式

每次生成 PPT 后，逐条检查。发现问题后：
1. 修复代码
2. 重新生成
3. 重新检查
4. 确认所有检查项通过后交付

> 此标准可迭代更新。发现新的常见错误后补充到对应类别。
