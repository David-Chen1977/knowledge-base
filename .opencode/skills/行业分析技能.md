# 行业分析技能

当我加载此技能时，我会按以下框架执行行业/公司分析。

## 输入要求

用户在发起分析任务时需尽可能提供：
- 赛道/公司名称
- 分析目的（投资决策/跟踪研究/竞品对标）
- 关注重点（技术/政策/财务/竞争格局）
- 时间范围

## 分析框架

### 1. 信息收集阶段

- 并行搜索：行业新闻、研报摘要、公司公告、财务数据
- 信息源优先级：SEC  filings / 公司官网 / 权威媒体 / 行业垂直媒体 / 学术论文
- 提取关键数据：市场规模、增速、竞争格局、核心玩家财务指标

### 2. 结构化分析

输出结构如下：

```
# [赛道/公司] 分析报告
## 一、核心结论（300 字以内）
## 二、行业概览
   - 产业链图谱
   - 市场规模与增速
   - 政策环境（国内 + 海外）
## 三、竞争格局
   - 主要玩家及市场份额
   - 差异化竞争要素
   - 集中度变化趋势
## 四、重点标的分析（如适用）
   - 财务分析（营收/利润/增速/估值）
   - 核心壁垒
   - 风险点
## 五、催化剂与风险
   - 短期催化剂（3-6 个月）
   - 中长期趋势（1-3 年）
   - 主要风险因素
## 六、信息来源
```

### 3. 输出格式

- 默认输出 Markdown 格式，存放于项目文件夹 `_output/` 目录
- 如需 Word/PDF，可额外生成
- 每个数据点标注来源

## 迭代机制

- 用户对分析的反馈会记录到项目 `README.md` 的"迭代记录"部分
- 我发现的新的优质信息源会持续纳入搜索优先级
