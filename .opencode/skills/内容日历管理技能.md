# 内容日历管理技能

当我加载此技能时，我会按以下流程管理内容排期。

## 依赖工具

- `schedule_manager.py` — 在 `/Volumes/移动硬盘/工具包与模板/内容管线/`
- `topic_sense.py` — 选题感知引擎（同目录）
- `pipeline.py` — 主线排器（同目录）
- `内容日历/README.md` — 日历源文件

## 工作流程

### 1. 查看当前排期

```
python3 schedule_manager.py status     # 查看完整日历排期
python3 schedule_manager.py due        # 查看本周到期内容
python3 schedule_manager.py recommend  # 推荐本周最佳选题
```

### 2. 推进排期

当用户确认新一周内容后：
- 更新 `内容日历/README.md` 中的进度追踪表格
- 标记已完成项（将 `📝` 改为 `✅`）
- 如需要，添加新排期行

### 3. 选题管理

```
python3 topic_sense.py scan     # 扫描知识库缺口，生成新选题
python3 topic_sense.py show     # 查看当前选题池
python3 pipeline.py start t001  # 开始生产一个选题
```

### 4. 日历更新规则

- 内容日历文件: `OpencodeWorkspace/内容日历/README.md`
- 排期变更必须同步更新 README 中的两个表格：
  1. **季度规划表格**（周次、主题、类型、四件套、依赖、难度）
  2. **进度追踪表格**（主题、公众号、网站、PPT、视频、状态）
- 每次更新后运行 `schedule_manager.py status` 确认排期一致

## 迭代记录

- 用户确认的新主题 → 写入选题池（topic-pool.json）
- 完成的内容 → 日历中标记 ✅
- 取消或延期的内容 → 移至待定选题池底部
