# 协议审查技能

当我加载此技能时，我会按以下框架协助审查协议文本。

## 输入要求

- 协议原文（Word/PDF/文本格式均可）
- 审查目的（投资协议/NDA/对赌/合伙协议）
- 你的立场（投资方/融资方）
- 特别关注条款（如有）
- 风险偏好（保守/中等/激进）

## 审查框架

### 1. 全文解析

- 识别协议类型和核心交易结构
- 提取关键当事人、金额、时间节点
- 标注条款完整度（有无缺失关键条款）

### 2. 条款逐项审查

按以下优先级逐条分析：

```
🔴 高风险（必须修改）
  - 赔偿责任条款（赔偿范围、上限、期限）
  - 对赌/业绩承诺条款
  - 控制权变更条款
  - 竞业禁止条款
  - 争议解决/管辖法律
  
🟡 中风险（建议争取修改）
  - 陈述与保证条款（范围、存续期）
  - 交割条件
  - 分红/利润分配条款
  - 退出机制
  - 信息权/检查权
  
🟢 低风险（了解即可）
  - 定义条款
  - 通知条款
  - 可分割性
  - 完整协议条款
```

### 3. 输出格式

```
# 协议审查报告
## 一、基本信息（协议类型、各方、金额、日期）
## 二、总体评价（1-2 段核心判断）
## 三、条款审查明细
   - [条款编号/名称] | [风险等级] | [分析] | [修改建议]
## 四、谈判建议（按优先级排序的谈判要点）
## 五、术语表（关键定义的简释）
```

### 4. 对比功能（如提供多个版本）

- 逐条款对比前后差异
- 标注修改方和修改方向（对谁有利）
- 量化评估修改影响

## 隐私说明

- 所有协议文本仅用于本次分析，不会保留或外传
- 如需长期参考，在项目文件夹内创建 `_input/` 和 `_draft/` 目录管理版本
