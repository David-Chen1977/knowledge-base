# 公众号文章质量控制规范 v1.0

> 从选题到发布的标准化质量关卡。**每篇文章必须逐项自检通过后方可发布。**

---

## 一、标题规范

| # | 检查项 | 标准 | 通过 |
|---|--------|------|------|
| 1 | 字数 | ≤15字，极端情况不超过18字 | □ |
| 2 | 不概括 | 不试图用标题说清全文，留好奇心缺口 | □ |
| 3 | 不拼凑 | 不用"XX，YY"两个半句拼成长标题 | □ |
| 4 | 不夸大 | 不用"疯狂/震惊/史诗/百年一遇/所有人都在忽视" | □ |
| 5 | 不堆数据 | 标题里不塞数字 | □ |
| 6 | 口语化 | 读起来像一个人在说话，不像新闻标题 | □ |
| 7 | 有态度 | 要么有观点、要么有疑问、要么有冲突 | □ |
| 8 | 贴内容 | 读者看完标题点进来，内容要对得上期待 | □ |

**参考写法：**

| ✅ 推荐 | ❌ 不推荐 |
|---------|----------|
| "AI用电，账还没算" | "四部门发布行动方案，AI与能源双向赋能迎来政策红利期" |
| "政策到了，钱还没到" | "2026年储能赛道最大的预期差在哪？——写在四部门文件出台之际" |
| "绿电等到了算力吗" | "AI训练使数据中心用电量激增，绿电直连政策升级打开千亿市场" |
| "避开这五种公司就够了" | "液冷赛道深度研究：从技术路线到标的筛选的全景分析" |

## 二、篇幅与内容深度

| # | 检查项 | 公众号版 | 网站版 |
|---|--------|---------|-------|
| 1 | 总字数 | 2000-3500字 | 3000-5000字 |
| 2 | 开篇钩子 | ≤200字抓住注意力 | ≤300字抛出矛盾 |
| 3 | 个人判断 | ≥2处明确立场 | ≥3处明确判断 |
| 4 | 数据支撑 | ≥10个具体数据点 | ≥20个具体数据点 |
| 5 | 案例/场景 | ≥2个 | ≥3个 |
| 6 | 框架归纳 | ≥2个框架 | ≥3个框架 |
| 7 | 行动指引 | 至少1条可执行建议 | 分层建议（同行/创业者/自己） |
| 8 | 免责声明 | ✅ 必须 | ✅ 必须 |

---

## 三、排版与格式

| # | 检查项 | 标准 |
|---|--------|------|
| 1 | 段落长度 | 每段≤5行，平均3行。一个观点一段 |
| 2 | 短句为主 | 单句≤40字，长句拆成短句 |
| 3 | 引用块 | 关键判断/核心数据用 `> ` 引用块突出 |
| 4 | 序号列表 | ≥3项的并列内容用数字编号(1. 2. 3.) |
| 5 | 无序列表 | 并列属性用短横线 `-` 列表 |
| 6 | 加粗 | 核心观点/数据用 `**` 加粗，每篇≤10处 |
| 7 | 分隔线 | 主要板块之间用 `---` 分隔 |
| 8 | 表格 | 数据对比/标的映射用表格，≤7列 |
| 9 | 标题层级 | 最多三级：`###` > `####` > `**` |
| 10 | 留白 | 标题前后空行，段落之间空行 |
| 11 | 数据来源 | 每个关键数据标注来源/截止日期 |
| 12 | 封面图 | 1080×606px，与主题相关 |

## 四、内容结构模板

### 公众号版结构

```
开篇钩子（场景/对话/数据冲击）
  → 核心判断（一句话立场）
  → 背景展开（政策/市场/技术驱动）
  → 框架拆解（2-3个框架，每个300-500字）
  → 深度分析（数据+案例+个人判断）
  → 标的映射（表格）
  → 行动指引（分层建议）
  → 免责声明
  → 互动引导
```

### 网站版结构

```
开篇（故事/矛盾/数据冲击）
  → 背景展开（政策全景）
  → 逻辑推演（层层递进，每个层次500-800字）
     ├── 框架一
     ├── 框架二
     └── 框架三
  → 个人判断（明确立场，≥3处）
  → 标的映射（表格+分析）
  → 行动指引（对同行/创业者/自己的分层建议）
  → 相关阅读链接
  → 免责声明
```

## 五、写作风格红线

### 必须做到

- [ ] 不说自己不信的话
- [ ] 不懂就说不知道，错了就认
- [ ] 用比喻代替术语
- [ ] 观点鲜明，不骑墙
- [ ] 没有废话，每句话都想过了

### 禁止

- ❌ 无个人判断的客观综述
- ❌ 只有观点没有数据
- ❌ 正文重复标题（公众号常见错误）
- ❌ 没有免责声明
- ❌ 各渠道简单复制粘贴（必须差异化）
- ❌ "我早就说过""所有人都在忽视"类自大表述
- ❌ 暴富叙事、确定性价格预测
- ❌ 数据无来源、无法核查

## 六、发布前检查清单

**每篇文章发布前逐项检查：**

- [ ] 标题 ≤15字、无夸大、有好奇心缺口
- [ ] 开篇200字有钩子（场景/对话/数据冲突）
- [ ] 总字数 2000-3500（公众号）/ 3000-5000（网站）
- [ ] 个人判断 ≥2处（公众号）/ ≥3处（网站）
- [ ] 核心数据 ≥10个，有来源标注
- [ ] 每段≤5行，平均3行
- [ ] 引用块 ≥1处突出关键判断
- [ ] 表格 ≥1个（数据对比或标的地图）
- [ ] 分段合理，板块间用`---`分隔
- [ ] 行动指引 ≥1条
- [ ] 免责声明 ✅
- [ ] 封面图已准备（1080×606）
- [ ] 朗读一遍，语感顺畅
- [ ] 各自媒体版本有差异（非简单复制）
- [ ] 数据截止日期已标注
- [ ] 无重复标题
