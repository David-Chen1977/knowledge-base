# 内容管线工作流 · SOP

**版本：** v2.0 | **更新：** 2026-06-28
**适用：** 从选题到发布的全流程标准操作程序

---

## 一、管线总览

```
选题触发 → 素材准备 → 写作 → 多格式输出 → 发布 → 回流闭环
   ①          ②         ③        ④         ⑤       ⑥
```

| 阶段 | 耗时 | AI 角色 | 人工角色 |
|------|------|---------|---------|
| ① 选题触发 | 5min | 扫描知识库+日历，提选题建议 | 确认选题方向 |
| ② 素材准备 | 15min | 知识库提取+websearch 补充最新数据 | 提供额外信源/判断 |
| ③ 写作 | 30-60min | 初稿（按风格指南） | 调整语气/角度/深度 |
| ④ 多格式输出 | 20min | 同时生成公众号版+网站版+PPT | 精修重点内容 |
| ⑤ 发布 | 10min | 公众号草稿箱推送（wenyan-mcp） | 终审→点击发布 |
| ⑥ 回流闭环 | 10min | 回流记录+日历更新+问题跟踪更新 | — |

**目标：从选题到发布 2 小时内完成一篇公众号文章。**

---

## 二、各阶段详细流程

### 阶段 ① 选题触发

**触发源（按优先级）：**

```
P0: 当前关注.md P0 变化
  → 立即评估是否需要插队产出
  → 示例：算电协同设为 P0 → 算电协同系列提前产出

P1: 内容日历排期到期
  → 按日历推进，提前 3 天准备素材
  → 每周日检查下周排期

P2: 待定选题池 → 正选提升
  → 当 P0/P1 选题全部完成时，从池中提一个
  → 优先选与当前关注方向最近的主题

P3: 突发热点
  → 评估是否值得插队（标准：与投资赛道相关+72h内有时效性）
  → 如值得，加入排期最前面
```

**选题确认标准（必须全部满足）：**

- [ ] 与投资赛道（电力新能源/AI）相关
- [ ] 有足够的判断空间（不是数据堆砌）
- [ ] 知识库有基础素材（或可快速补充）
- [ ] 读者会在意（有行动指引价值）

### 阶段 ② 素材准备

```markdown
## 素材清单模板

### 知识库提取
- [ ] 阅读 `知识库/[赛道]/` 相关文件
- [ ] 提取：关键数据、技术路线对比、标的公司、估值参考

### 最新数据补充（websearch）
- [ ] 搜索最近 1 个月的政策变化
- [ ] 搜索最新融资/交易动态
- [ ] 搜索关键数据更新（市场规模、装机量等）

### 个人判断准备
- [ ] 你的核心观点是什么？
- [ ] 和三方不同的看法是什么？
- [ ] 对投资人有何行动建议？
```

### 阶段 ③ 写作（按风格指南）

> ⚠️ v2.0 原则：公众号版不再压缩字数，直接采用网站深度版全文（~3000字）。
> 实践证明压缩版损失深度和冲击力，读者感知不到价值。两条管线共用同一正文，
> 差异仅在于标题和开篇处理方式（见下方标注）。

**公众号版（~3000字，同网站深度版）：**
```
标题（≤15字，必须有勾子，与内容直接关联，不拼凑）
开篇钩子（2-3句抓人，从反常观察/具体场景切入）
  → 核心判断（一句话立场）
  → 逻辑推演（层层递进，每个论点有数据）
  → 个人判断（有立场，≥3处）
  → 行动指引（对投资人的建议）
  → 免责声明
  → ❌ 不写"点赞在看转发"
```

**网站深度版（2000-4000字）：**
```
标题（可与公众号不同，更偏知识性）
开篇钩子（故事/矛盾/问题）
  → 背景展开（数据+事实铺陈）
  → 逻辑推演（层层递进）
  → 个人判断（有立场）
  → 行动指引（对投资人的建议）
  → 免责声明
```

**写作检查：** 参照 `知识库/写作风格参考-陈道雷.md` §五 检查清单

### 阶段 ④ 多格式输出

**并行产出策略：**
```
同一主题 → 按平台分别产出独立内容，不可简单搬运。

公众号版（~3000字，同网站深度版正文）
  标题必须有勾子（≤15字，数据+反常/冲突最佳）
  开篇处理更口语化，段落更短
  正文不做任何删减

网站深度版（2000-4000字）
  标题可更偏知识性，不强制勾子
  所有数据+分析+框架

头条版（原创独立写作）
  ⚠️ 头条要求"原创首发"，不可与公众号/知乎内容重复
  标题更直接、更有冲击力，可用数字+结论结构
  段落短，每段不超过3-4行
  开头必须有强钩子（冲突/反差/惊人数据）
  语气更贴近大众读者，减少专业术语密度
  结尾可直接引导讨论/互动

知乎版（原创独立写作）
  ⚠️ 知乎同样要求"原创首发"
  标题偏知识型，严谨不浮夸
  结构更完整：背景→分析→判断→开放问题
  语气专业但有温度，适合深度阅读场景
  可加入更多技术细节和逻辑推演
  结尾以开放性问题收束，引导讨论

PPT（15-25页）
  标题+关键数据+投资图谱
  目前用 nmg-ppt 管线（deck.json → native_pptx.py）

❗ 跨平台原则（已固化 2026-06-28）：
  - 各平台版本必须有实质性差异（角度/结构/语气/案例），不可仅做格式微调
  - 先发平台的版本不可直接用于后发平台
  - 同一选题产出顺序：公众号/网站 > 头条 > 知乎（或按需调整）
  - 每篇文章至少有 1 个该平台独有的切入角度或数据点
```

### 阶段 ④-A：质量控制门（MUST PASS）

**每篇文章发布前必须通过 `公众号质量控制规范.md` 的自检清单。**

关键检查项（详见质控规范全文）：
- [ ] 标题 ≤15字、不夸大、有好奇心缺口
- [ ] 总字数 2000-3500（公众号）/ 3000-5000（网站）
- [ ] 开篇200字有钩子（场景/对话/数据冲突）
- [ ] 个人判断 ≥2处（公众号）/ ≥3处（网站）
- [ ] 核心数据 ≥10个，有来源标注
- [ ] 每段≤5行，排版有引用块/表格/列表
- [ ] 行动指引 ≥1条 | 免责声明 ✅
- [ ] 封面图已准备（1080×606）
- [ ] 各自媒体版本有差异（非简单复制粘贴）

> ⚠️ **质量控制规范位于：** `知识库/内容日历/公众号质量控制规范.md`
> 首次发布前须逐项手动检查。已通过的检查项保留为 ✅，未通过的列在迭代修复清单中。

### 阶段 ⑥ 回流闭环

每次产出后必须：

```
1. 创建回流文件
   → `知识库/[赛道]/_回流_YYYY-MM-DD_类型.md`
   → 记录：新增数据、关键判断、交叉引用

2. 更新日历进度
   → `内容日历/README.md` 进度追踪表
   → 对应主题标 ✅

3. 更新内容产出索引
   → `知识库/内容日历/README.md`
   → 新增产出记录

4. 检查待深化问题
   → 如果回答了某个问题 → 登记到"回答记录"
   → 如果发现了新问题 → 新增到清单

5. 触发下一选题
   → 检查 `当前关注.md` P0 是否有变化
   → 确认下周排期
```

## 三、质量标准

### 必须达到

| 维度 | 公众号版 | 网站版 |
|------|---------|-------|
| 标题勾子 | ✅ ≤15字，数字+反常结构优先，与内容直接关联 | ✅ 可更偏知识性 |
| 开篇钩子 | ✅ 2-3句抓人 | ✅ 故事/矛盾/问题 |
| 正文 | ✅ 不压缩（~3000字），同网站版 | ✅ 完整版 |
| 个人判断 | ✅ 明确立场（≥3处） | ✅ 明确的判断 |
| 数据支撑 | ✅ 10+个 | ✅ 15-30个 |
| 框架归纳 | ✅ 2-3个框架 | ✅ 3-5个框架 |
| 行动指引 | ✅ 分段建议 | ✅ 分段建议 |
| 免责声明 | ✅ 必须 | ✅ 必须 |
| 写作风格匹配 | ✅ 参考风格文件 | ✅ 参考风格文件 |

### 禁止

- ❌ 无个人判断的客观综述
- ❌ 只有观点没有数据
- ❌ 正文重复标题（公众号常见错误）
- ❌ 没有免责声明
- ❌ 各渠道简单复制粘贴（必须差异化）
- ❌ ❌ 头条/知乎使用公众号已发内容直接搬运（违反原创首发规则）
- ❌ 同一篇文章在不同平台仅改标题和首段（平台间必须有实质性差异）

## 四、异常处理

| 场景 | 处理方式 |
|------|---------|
| 选题素材不足 | 降到待定选题池，换一个主题 |
| 写作风格偏离 | 对照风格参考检查清单逐项修正 |
| 公众号发布失败 | 保存为 Markdown，手动复制到微信编辑器 |
| PPT 生成出错 | 先用大纲+关键数据手动做，后续修复脚本 |
| 临时热点插队 | 加速流程：跳过素材准备阶段，直接写作 |
