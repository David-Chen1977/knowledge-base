# OpencodeWorkspace 工作准则

---

## 一、文件夹结构

```
OpencodeWorkspace/
├── .opencode/                 # OpenCode 配置（技能、规则）
│   └── skills/                # 自定义技能
├── _archive/                  # 已完成的归档项目
├── _templates/                # 项目模板文件
├── ANL_xxx_YYY_YYYYMMDD/      # 行业分析项目
├── AGR_xxx_YYY_YYYYMMDD/      # 协议审查项目
├── PPT_xxx_YYYYMMDD/          # 路演/PPT 项目
├── WECHAT_xxx_YYYYMMDD/       # 公众号文章项目
├── VIDEO_xxx_YYYYMMDD/        # 短视频项目
└── PRJ_xxx_YYYYMMDD/          # 通用项目
```

---

## 二、项目文件夹命名规范

### 命名格式

```
[类型前缀]_[项目简述]_[日期YYYYMMDD]
```

### 类型前缀

| 前缀 | 含义 | 典型场景 |
|------|------|----------|
| `ANL` | Analysis / 行业分析 | 赛道研究、公司尽调、竞争格局 |
| `AGR` | Agreement / 协议审查 | 投资协议、NDA、对赌条款 |
| `PPT` | Presentation / 路演 | 基金路演、投资人汇报 |
| `WECHAT` | 公众号文章 | 行业观点、投资思考 |
| `VIDEO` | 短视频 | 脚本、分镜、素材 |
| `PRJ` | 通用项目 | 不属于上述分类的工作 |

### 示例

```
ANL_固态电池产业链_20260610
AGR_某资本_对赌协议_20260609
PPT_新能源一期基金_20260608
WECHAT_光伏板块波动分析_20260607
VIDEO_储能赛道科普_20260606
```

### 规则

1. 项目简述使用中文，控制在 10 字以内
2. 日期统一使用 `YYYYMMDD` 格式
3. 不同单词之间用下划线 `_` 连接
4. 项目文件夹内至少包含一个 `README.md` 说明本项目背景和产出

---

## 三、项目文件夹内部结构（推荐）

```
ANL_xxx_YYYYMMDD/
├── README.md                    # 项目背景、结论摘要
├── _input/                      # 输入材料（原文、数据源）
├── _output/                     # 最终产出（报告、协议、PPT）
├── _draft/                      # 过程草稿、中间版本
└── _ref/                        # 参考资料、相关链接
```

可根据项目类型精简，但 `README.md` 和 `_output/` 建议保留。

---

## 四、协作原则

### 对 AI（我）的要求

1. **所有文件操作默认在 `OpencodeWorkspace/` 下进行**，除非你明确指定其他路径
2. **每个项目必须创建独立文件夹**，不得散落在根目录
3. **产出标准格式**：分析报告用 Markdown，正式文档可转 Word/PDF，PPT 用 `.pptx`
4. **每次交付附带说明**：做了什么、用了什么信息源、建议下一步做什么
5. **主动归档**：项目完成后主动询问是否需要移入 `_archive/`
6. **持续学习**：记录你的反馈并调整行为模式

### 对你（人类）的要求

1. **给项目取好名字**：按规范命名，或在开始时告诉我主题我来命名
2. **提供背景**：每次新任务至少给 2-3 句话说明上下文
3. **给反馈**：告诉我输出好不好，不好在哪里
4. **提供样本**：想让我模仿某种风格，就给我参考材料

---

## 五、版本与归档

1. **进行中的项目**：放在根目录下
2. **已完成的项目**：移入 `_archive/`，按年份分子文件夹
3. **关键交付物**：我会在 `README.md` 中记录每次修改要点和版本变化

---

## 六、`.opencode/` 目录说明

```
.opencode/
└── skills/          # 自定义技能
```

- 技能是对特定工作流的封装，可用 `skill` 工具加载
- 每个技能是一个 `.md` 文件，包含特定任务的指令模板
- 我会根据你的工作流持续创建和优化这些技能
