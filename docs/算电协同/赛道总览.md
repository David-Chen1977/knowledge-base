# 算电协同 · 赛道总览

**定义：** 算电协同（Computing-Electricity Coordination）指通过协调计算负载与电力系统的时空分布，实现算力需求与电力供给的最优匹配。核心目标：**让电等算力，还是算力等电？**

**成熟度：** 早期成长期（2025-2026 政策驱动元年）
**PE/VC 重心：** 数据中心需求响应、算力调度平台、液冷散热、AI+电力交易

---

## 一、核心逻辑

### 为什么算电协同现在重要？

```
算力爆发 → 数据中心用电量激增 → 电网承受压力
           ↓
    算电协同 → 把数据中心从"电老虎"变成"柔性负荷"
           ↓
    收益：电网减负 × 数据中心降本 × 绿电消纳
```

**关键数据：**
- 2026年中国数据中心用电量预计 ~3000亿 kWh，占全社会用电量 ~3.5%
- 2025-2030 数据中心用电 CAGR 15-20%（AI 训练+推理驱动）
- 单座千P智算中心年用电量 ≈ 3-5 亿 kWh（相当于一个小型工业园）
- 数据中心参与需求响应可降低峰值负荷 10-30%

### 核心矛盾

| 矛盾 | 说明 |
|------|------|
| **时间错配** | AI 训练夜间低谷→电价低，推理白天高峰→电价高 |
| **空间错配** | 算力集中在东部→电从西部来→输电损耗+通道瓶颈 |
| **利益错配** | 电网要稳定、数据中心要低价、政府要碳排——三方博弈 |

## 二、驱动因素

### 2.1 政策驱动（最强）

| 政策 | 时间 | 要点 |
|------|------|------|
| 算电协同纳入**政府工作报告** | 2026 | 首次写入国家级规划 |
| 电改 2.0 | 2025-2027 | 电力现货市场全覆盖，数据中心可参与 |
| 数据中心能效标准 (PUE<1.25) | 2025 | 新建数据中心强制要求 |
| "东数西算"二期 | 2025-2028 | 算力向西迁移 |
| 新型电力系统建设 | 2024-2030 | 源网荷储一体化 |

### 2.2 经济驱动

- 数据中心用电成本占运营成本 30-60%
- 参与需求响应可降低电费 10-20%
- 绿电直购可降低碳成本+品牌溢价
- 液冷散热比风冷节能 30-50%（PUE 从1.4→1.1）

### 2.3 技术驱动

- AI 调度算法成熟（DeepSeek/AI Agent 应用于电力调度）
- 液冷散热技术商用化（冷板式→浸没式）
- 储能成本持续下降（2026年LFP系统成本<0.5元/Wh）
- 虚拟电厂技术平台化

## 四、六大赛道详解

### 4.1 数据中心 + 需求响应（最快落地）

**逻辑：** 数据中心作为柔性负荷参与电力需求响应市场

**模式：**
- 削峰：电价高时降低算力负载（训练任务可中断/迁移）
- 填谷：电价低时增加算力负载（批量训练/数据备份）
- 调频：秒级响应电网频率波动

**代表事件：**
- 2025年内蒙古首批数据中心参与调峰辅助服务市场
- 2026年长三角启动数据中心虚拟电厂试点

**标的：** 光环新网、数据港、秦淮数据、万国数据

### 4.2 算力西迁 + 绿电消纳

**逻辑：** 把算力中心建在新能源富集区（西部），就地消纳绿电

**模式：**
- 源网荷储一体化：风电/光伏 + 储能 + 算力中心
- 绿电直购：数据中心直接与新能源场站签PPA

**挑战：**
- 网络延迟（推理场景受影响）
- 人才配套（西部运维人才短缺）
- 水资源（西部缺水，液冷需水量大）

### 4.3 AI + 电力调度

**逻辑：** AI Agent 自主优化电力系统调度

**方向：**
- 新能源出力预测（风光+负荷）
- 现货市场报价策略优化
- 电网故障自愈
- 虚拟电厂聚合调度

**标的：** 瓦特智能、清大科越、电享科技、浙达能源

### 4.4 液冷散热

**逻辑：** 散热占数据中心能耗 30-40%，液冷是降PUE的关键

**路线：**
- 冷板式液冷（已商用，PUE<1.2）
- 浸没式液冷（早期，PUE<1.05，但维护复杂）

**标的：** 英维克、高澜股份、网宿科技（绿色云图）

### 4.5 虚拟电厂（VPP）聚合数据中心

**逻辑：** 把分散的数据中心聚合为虚拟电厂，统一参与电力市场

**壁垒：**
- 聚合平台技术（实时调度+AI优化）
- 客户资源（签约数据中心数量）
- 电力市场交易资质

### 4.6 低功耗算力芯片

**逻辑：** 同等算力下功耗更低→电费更少→PUE压力更小

**方向：**
- 推理专用芯片（ASIC/NPU）
- 存算一体芯片（减少数据搬运功耗）
- 光子计算（长期）

## 六、风险与挑战

1. **商业模式不成熟** — 需求响应收益机制还在试点，谁出钱？
2. **政策不确定性** — 电力市场化改革进度可能不及预期
3. **技术整合难度** — IT 和 OT 的融合需要跨领域人才
4. **利益博弈** — 电网、数据中心、政府三方博弈可能延缓落地
5. **资源限制** — 西部算力中心的水资源、网络、人才配套不足
