# 算电协同进入落地期：从算力用户的需求倒推，投资机会在哪里

**2026-06-12 | 陈道雷**

> 算电协同的最终落脚点不是电厂、不是电网、不是数据中心——而是为算力用户提供更低成本、更稳定、更绿色的算力。
>
> 一切策略选择，最终都取决于用户的需求。

---

## 引子：一个被忽略的最关键问题

关于算电协同的讨论大部分围绕供给侧——绿电多少钱、液冷什么PUE、哪个省政策好。

但很少有人认真问一个问题：**算力用户到底需要什么？**

这里说的"用户"，是那些真正在跑AI训练和推理、在部署大模型的企业。他们不关心绿电怎么买的、不关心液冷是冷板式还是浸没式、不关心各省的能耗指标政策差异——他们只关心三件事：

1. **算力贵不贵？**
2. **响应快不快？**
3. **供应链稳不稳？**

这三件事，决定了算电协同所有策略选择的底层逻辑。

---

## 一、从用户需求出发：三类算力负载，三种完全不同的要求

算力不是铁板一块。不同的工作负载，对时延、成本、可靠性的要求天差地别。

### 第一类：AI训练

**特征：** 大规模、高并发、持续数周
**典型用户：** 大模型公司、科研机构

**需求画像：**
- 对时延最不敏感（可以接受>20ms）
- 对成本最敏感（训练成本直接决定模型迭代速度）
- 对稳定性要求高（一次训练中断损失几十万）
- 不需要跟终端用户交互

**→ 选址偏好：** 电价最低的地区优先，西部能源富集区天然匹配

### 第二类：AI推理——在线

**特征：** 单次请求小、实时性要求高、持续运行
**典型用户：** AI应用（智能客服、代码助手、AI搜索）

**需求画像：**
- 对时延极度敏感（要求<10ms，甚至<5ms）
- 对成本中度敏感
- 需要靠近用户分布
- 扩缩容频繁，需要弹性

**→ 选址偏好：** 必须靠近一线城市或区域中心，分布式部署

### 第三类：AI推理——离线/批量

**特征：** 大批量、定时触发、延迟容忍
**典型用户：** 数据标注、视频转码、科学计算

**需求画像：**
- 对时延中等敏感（可接受50-100ms）
- 对成本非常敏感
- 可以集中也可分散

**→ 选址偏好：** 电价低+有一定网络基础设施的地区

### 一个核心洞察

**同一家AI公司，同时需要这三类算力。** 一家大模型公司，训练在西部、推理在边缘、批量处理可以两地跑。

这意味着：**算电协同不是"选一个地方"的问题，是"建一张网"的问题。** 单一节点的成本最优不等于全局最优，用户需要的是一个能同时满足三类负载的算力网络。

## 二、从需求到选址：六个关键决策变量

把用户需求翻译成选址逻辑，需要拆解六个变量：

### 变量1：时延约束——不可妥协的硬限制

这是最重要的筛选条件，没有之一。

| 应用场景 | 最大容忍时延 | 最大覆盖半径 | 可选区域 |
|---------|-------------|-------------|---------|
| 自动驾驶推理 | <5ms | ~100km | 一线城市城内 |
| 交互式AI（语音/对话） | <10ms | ~500km | 一线+强二线 |
| AI代码补全 | <15ms | ~800km | 主要城市群 |
| AI训练 | >20ms可接受 | 全国 | 西部地区 |
| 离线批量处理 | >50ms可接受 | 全国 | 不限 |

**实操含义：** 想做推理场景的算电协同项目，选址几乎没有弹性——用户在哪，算力就得在哪。只有训练场景才有"把算力搬到电便宜的地方"的选择空间。

### 变量2：电费成本——最重要但非唯一的成本项

电费占数据中心运营成本的30-60%，但**不等于综合成本**。用户算的不是电价，是TCO（总拥有成本）：

```
综合算力成本 = 电费 + 机房租金 + 网络带宽 + 运维人力 + 碳成本
```

西部电价低，但网络带宽成本高（数据传到东部有通道费）、运维人力贵（人才稀缺需要高薪），这些隐性成本可能吃掉电价优势的一半以上。

| 成本项 | 西部节点 | 东部节点 |
|-------|---------|---------|
| 电价（元/kWh） | 0.2-0.3 | 0.6-0.8 |
| 带宽及数据传输 | 高（需跨省通道） | 低（本地接入） |
| 运维人力 | 高薪难招 | 丰富但贵 |
| 碳成本 | 绿电充足，几乎为零 | 需购买绿证或碳配额 |

**实操含义：** 只在电价差足够大（>0.3元/kWh）且负载对时延不敏感的场景下，"西迁"才划算。对于中低时延敏感的推理场景，分布式部署的综合成本反而更低。

### 变量3：绿电与碳约束——从"加分项"变成"必选项"

越来越多的AI公司有明确的碳中和承诺。对于准备出海或服务跨国客户的企业，Scope 2（用电碳排放）直接决定能否接到订单。

这意味着：**绿电不再是"企业社会责任"，直接变成了客户要求。**

用户的选择逻辑很简单：
- 同样价格→选绿电比例高的
- 绿电贵一点→算碳成本账，如果碳配额购买成本比绿电溢价高，仍然选绿电
- 产品出口欧盟→必须满足碳边境调节机制（CBAM），绿电是刚需

**实操含义：** 绿电占比高的算力节点，获得了对特定客户群体的差异化定价权。这不是成本问题，是市场准入问题。

### 变量4：政府政策——正反两面都在影响决策

政策对用户选择的影响是双向的：

**正面（吸引力）：**
- 能耗指标优先保障（西部各省都在抢智算中心项目）
- 绿电补贴/优惠电价（内蒙古等地对数据中心有专项电价）
- 税收优惠（西部大开发政策，15%企业所得税）

**反面（限制力）：**
- 一线城市严格控制新增数据中心（北京、上海几乎不再批新建）
- 能效标准门槛提高（新建PUE必须<1.25）
- 碳配额逐年收紧

**实操含义：** 政策的影响不亚于技术和成本。在审批受限的城市，有"存量机房资源"的团队拥有结构性优势——不需要批文，改造就能用。

### 变量5：交付时间——最被低估的决策变量

这个变量很少被写进研报，但在实际决策中权重极高。

**新建一个智算中心需要多久？**

从拿地、审批、设计、建设、设备安装调试到投产——**2-3年是正常节奏，4年也不罕见。**

而客户的需求是什么样的？

2025年到2026年，AI算力需求年增长50-80%。一家AI公司今天需要的算力，半年后可能翻倍。**等不起。**

这就是为什么"存量盘活"模式在市场上获得如此高的关注度——不是因为它技术最优，是因为它**最快**。

| 交付方式 | 典型周期 | 制约因素 | 适合场景 |
|---------|---------|---------|---------|
| 存量机房改造 | 3-6个月 | 原机房承重/电力容量/层高 | **当前急需**的算力缺口 |
| 分布式算力仓 | 1-3个月 | 场地条件+设备供应 | **即时响应**的弹性需求 |
| 新建智算中心 | 2-4年 | 审批+建设+设备交付 | **长期规划**的大规模算力 |

**但"新建慢"不等于"新建没价值"。** 有两个判断值得注意：

**第一，存量改造的天花板是存在的。** 存量机房的建筑条件（层高、承重、电力增容空间）是物理约束。改造到一定程度，边际收益递减。当算力需求持续增长到存量资源无法承载时，新建是唯一选项。

**第二，客户的需求不是只有"现在就要"。** 成熟的企业在采购算力时，会同时做三件事：
- 这个月要用的算力→**租云/存量改造**（解决当下）
- 下个季度要用的算力→**分布式算力仓**（解决短期）
- 明年要用的算力→**规划新建智算中心**（解决长期）

这意味着：**新建和存量不是互斥的，是时间维度上的互补。** 旧改解决"等不起"的当下，新建投产时，旧改节点可以转向时延敏感或弹性负载。

### 变量6：供应链可靠性——最后一个，但最致命

2022年西部某数据中心因电网检修停电48小时，造成客户数十万元的训练中断损失。这种事件多了，用户对"电稳不稳"的关注度超过"电便不便宜"。

**用户的实际选择逻辑：**
- 双路供电是底线
- 有储能/柴油发电机备份是加分项
- 能接入多路电网（比如既有大电网又有新能源微网）是最优解

## 插曲：GPU短缺——一个加速行业洗牌的现实卡点

在分析框架之外，有一个正在发生的市场现实值得单独记录——**英伟达B300的严重缺货。**

B300（Blackwell B300）是当前最抢手的AI训练芯片。需求远超供给，导致了一系列连锁反应：

**第一，GPU缺货正在改变算力供给的竞争规则。**

以前建数据中心的逻辑是"先建好，等客户"。现在是"先锁定GPU供给，再建数据中心"。没有GPU的算力中心，建成也是空壳。

这意味着：**GPU供应链关系变成了算力供给的竞争壁垒。** 谁能从英伟达或OEM厂商拿到稳定的B300配额，谁就能优先锁定大客户的长协合同。没有配额，再低的电价、再快交付的机房，也只是一张空头支票。

**第二，缺货催生了市场的非理性行为。**

- 灰色市场溢价：B300的现货价格比官方定价高出50-100%
- 算力期货化：客户提前6-12个月下单锁定产能，价格在签约时锁定
- 国产替代加速：昇腾等国产芯片在B300缺货的窗口期获得了宝贵的验证和适配机会

**第三，这是算电协同投资中不可忽视的"变量X"。**

变量X不是技术问题、不是选址问题——它是**地缘政治+供应链格局**问题。美国对高端AI芯片的出口管制持续收紧，这种不确定性使得国内的算力基础设施规划面临一个结构性的难题：**即使你能在6个月内把机房改好，如果GPU到不了，算力就交付不了。**

这个变量对投资策略的影响：
- 有GPU供应链资源的团队（与英伟达/华为有稳定合作）比只有机房资源的团队更有竞争力
- 分布式推理场景对B300的依赖度低于训练场景（推理可以用多种芯片），这进一步强化了"边缘分布式路径更适合早期投资"的判断
- 国产芯片（昇腾、寒武纪）在训练场景逐步追赶的过程中，适配验证能力会成为算电协同项目的加分项

## 三、从变量到路径：三条落地路径的用户逻辑重述

在前面分析的基础上，重新审视三条落地路径，能更清楚地看到**每条路径本质上服务于哪类用户需求**：

### 路径一：西部集中式（新建）——服务于"长期战略+成本敏感"的用户

**时间定位：** 中长期（规划→投产 2-4年）

**典型用户画像：**
- 大模型预训练（需要大规模集群长期稳定运行）
- 有中长期战略规划的AI公司（提前锁定未来算力供给）
- 离线数据批处理、科学计算

**需求满足逻辑：**
```
用户需要：低电费 + 大规模集群 + 绿电覆盖 + 未来发展空间
    ↓
西部集中式新建：电价0.2元 + 千P级集群 + 绿电40%+ + 可扩展
    ↓
代价：2-4年建设周期，20ms+时延，运维人才稀缺
    ↓
结论：适合有提前规划能力的成熟客户，不适合"用急"的需求
```

**为什么客户会选择新建（尽管慢）：**
- 算力需求不是一次性的，是持续增长的曲线。今天的紧迫需求用旧改解决，但**明年、后年、五年后的需求，需要有战略性的基础设施来承接**
- 存量改造受建筑条件限制，无法无限制扩容。当企业规划千P级以上集群时，新建是唯一选项
- 新建可以从零设计，建筑规格、电力架构、冷却系统都是最优配比，长期运营效率高于旧改

**核心竞争壁垒：** 大资本开支能力、政府关系（拿地+能耗指标）、绿电直供能力

### 路径二：分布式边缘（以旧改为主）——服务于"时延敏感+现在就要"的用户

**时间定位：** 短期（1-6个月交付）

**典型用户画像：**
- 交互式AI应用
- 自动驾驶/机器人
- 企业私有化推理部署
- **一切"等不起"的算力需求**

**需求满足逻辑：**
```
用户需要：<5-10ms时延 + 弹性扩缩容 + 数据本地化 + 马上能用
    ↓
分布式边缘：靠近用户 + 存量机房盘活 + 1-3个月交付
    ↓
代价：单位电费较高、运维分散、单体规模有限
    ↓
结论：用户愿意为"快"和"近"支付溢价
```

**为什么这是当前最活跃的路径：**
- AI算力需求暴增的速度，远超新建周期——供需缺口必须由"快的方案"来填补
- 存量机房盘活绕过了审批瓶颈，是当下少数能快速扩大供给的方式
- 当未来新建节点投产时，这些旧改节点可以自然转向时延敏感负载或作为弹性缓冲

**核心竞争壁垒：** 存量机房资源获取能力、快速交付能力、本地运维网络

### 路径三：存量改造（节能升级）——服务于"已有资产提效"的用户

**时间定位：** 中短期（3-6个月改造周期）

**典型用户画像：**
- 运营商自有数据中心
- 企业自建机房
- 中型互联网公司

**需求满足逻辑：**
```
用户需要：增量算力但不新建 + 控制资本开支 + 提升能效
    ↓
存量改造：气流优化 + 液冷背板 + 综合能源调度
    ↓
代价：改造空间受原建筑条件限制
    ↓
结论：投入产出比最高的路径，但天花板可视
```

**核心竞争壁垒：** 节能技术改造方案、工程实施经验、性价比

## 四、投资人的视角：三条路径的变现逻辑

这是我作为投资人最关心的问题——**每条路径对应什么商业模式？谁来付费？能赚多少钱？**

### 路径一：西部集中式

| 维度 | 分析 |
|------|------|
| **谁付费** | AI公司（训练任务）、云厂商 |
| **商业模式** | 算力租赁（按卡/按时）、定制化机房托管 |
| **收入确定性** | 中高——锁定大客户后稳定 |
| **资本强度** | 极高——单项目数十亿 |
| **利润率** | 中——电费低但折旧高 |
| **资产属性** | 重资产，适合长期资本 |
| **退出通道** | 资产包出售/REITs |

**关键判断：** 适合产业资本和长期资金，不适合财务投资人的基金周期（基金5-7年可能等不到退出）。

### 路径二：分布式边缘

| 维度 | 分析 |
|------|------|
| **谁付费** | AI应用公司、企业客户 |
| **商业模式** | 算力零售（按API调用/按推理量）、边缘节点租赁 |
| **收入确定性** | 中——客户分散，单一客户依赖度低 |
| **资本强度** | 低——单节点百万级 |
| **利润率** | 中高——延迟溢价 |
| **资产属性** | 轻资产，可快速复制 |
| **退出通道** | 被收购（云厂商/运营商）或独立上市 |

**关键判断：** 最适合PE/VC入场的路径。门槛低、可复制、有网络效应——节点越多，对客户的吸引力越大，议价能力越强。

### 路径三：存量改造

| 维度 | 分析 |
|------|------|
| **谁付费** | IDC运营商、企业 |
| **商业模式** | EPC合同、节能分成、改造后运营分成 |
| **收入确定性** | 高——合同已签 |
| **资本强度** | 低到中 |
| **利润率** | 中——技术溢价 |
| **资产属性** | 项目制 |
| **退出通道** | 服务合同到期，持续性不确定 |

**关键判断：** 好生意，但不是大生意。天花板取决于改造团队规模，不是平台型机会。

## 五、综合判断：投资人应该怎么选

拉通看六个变量，我自己的排序是：

### P0：分布式边缘算力（立即关注）

**为什么？**
- 需求端：推理负载正在超过训练负载，时延敏感场景越来越多，而且需求"现在就要"
- 供给侧：存量机房盘活绕开了新建审批，3-6个月交付，正好匹配客户当前无法等待的需求
- 竞争格局：跑马圈地阶段，谁先拿到存量机房资源谁占优，先发优势明显
- 商业模式：轻资产、可复制、有网络效应——节点越多，对客户议价能力越强
- 资本效率：单节点百万级，适合PE/VC基金周期

### P1：西部集中式新建（布局，但选对合作方）

**为什么？**
- 需求确定：客户不只是要当下的算力，还在为明年、后年做规划
- 但入场门槛极高：资本强度数十亿级、需要强政府关系拿地拿指标、建设周期跨越基金周期
- 存量改造有天花板，当用户需要千P级集群时，新建是唯一选择
- 适合产业资本/国资平台牵头，财务投资人以参股或债权方式参与

**一个需要警惕的判断：** 如果只做新建不做旧改，等于只接了客户"长期需求"的其中一半。一个完整的算电协同投资组合，应该**同时覆盖短期交付能力和长期战略布局**。

### P2：存量改造技术服务（选择性参与）

**为什么？**
- 收入确定性强，但不是高速增长赛道
- 适合作为生态能力补充，不适合作为主赛道
- 如果团队有核心技术（气流优化仿真、液冷改造方案），可以作为差异化服务能力

## 六、未来场景推演：当技术迭代从线性变成指数

### 一个核心判断：AI时代≠互联网时代

这篇文章的分析框架——五个决策变量、三条路径、变现逻辑——都是基于当前可观察的行业现实推导出来的。

但有一个更大的问题需要回答：**这个框架能管多久？**

我的判断是：**AI时代的技术迭代节奏，跟过去二十年互联网时代的经验不可类比。** 用互联网时代的线性外推去理解AI时代，可能是最大的认知风险。

### 为什么不一样？

互联网时代（1995-2020）的技术演进大致遵循**线性可预测**的节奏：
- 芯片性能每18个月翻倍（摩尔定律）
- 带宽每12-18个月翻倍
- 应用层的创新节奏大体匹配基础设施的成熟节奏

AI时代的特征完全不同：

**技术迭代是指数级的，而非线性的。**
- 模型参数每12个月增长10倍
- 训练算力需求每6-12个月翻倍
- 推理成本每12个月下降一个数量级
- 新的技术范式（MoE、多模态、Agent）不是渐进改进，是跃迁

### 指数级变化对基础设施意味着什么？

**第一，今天认为"超前"的规划，三年后可能不够用。**

如果我们用线性思维做基础设施规划——假设算力需求年增长30-50%——那今天规划的千P级智算中心，3年后可能刚好填满。但实际需求很可能远超这个假设。

**这意味着"过度建设"的风险可能被高估，"建设不足"的风险被低估。**

**第二，技术迭代的速度可能持续领先于物理世界的建设速度。**

芯片升级周期12-18个月，软件迭代更短。但建一个数据中心需要2-4年。

这个时间差是结构性的——不是管理能解决的，是物理世界的建造速度跟不上数字世界的迭代速度。

**这意味着：交付速度本身就是稀缺能力。** 能3个月交付的团队，比能做出最优技术方案的团队更有竞争力。因为等你2年后建好最优方案，技术已经迭代了两轮。

**第三，基础设施从"成本中心"变成"战略瓶颈"。**

在互联网时代，算力是商品——不够了就买，云厂商随时提供。在AI时代，算力是稀缺资源——头部模型公司会提前锁定未来2-3年的算力产能。

这不是推测，已经发生了。2025年起，顶级AI公司与数据中心运营商签的都是"多年+数十亿"级别的长协。

### 在这个快速变化的世界里，有哪些可能的确定性？

我试着找了几条：

**确定性1：算力总需求将持续增长，而且增速可能超过大多数预测模型。**
这不是乐观，是逻辑——只要AI还在产生商业价值，企业就会持续投入更多算力。成本下降只会刺激更多应用，而不是减少总支出（杰文斯悖论）。

**确定性2：能源将成为算力的最终瓶颈。**
芯片可以越来越强，但电网的扩容速度是物理限制。当算力增长到一定程度，"能不能供上电"会比"有没有更好的芯片"先成为瓶颈。这也是算电协同这个赛道存在的前提。

**确定性3：基础设施的"交付速度"越来越重要。**
在指数级变化的世界里，6个月交付的方案比2年交付的"最优方案"更有竞争力。因为2年后你面对的市场和技术环境已经完全不同。

**确定性4：绿电和碳合规将成为算力的"准入门槛"。**
不是锦上添花，是上桌的入场券。出口型企业和服务跨国客户的AI公司，没有绿电覆盖的算力节点不在考虑范围内。

**确定性5：能够跨越"技术—工程—商业"三个层面的团队，稀缺性会越来越高。**
这个行业不缺懂技术的人，也不缺懂工程的人，但既懂技术演进方向、又能落地工程建设、还能设计商业模式的人——极为稀缺。

### 对投资人的启示

如果以上判断正确，那么：

**不要用"泡沫"的框架去理解AI时代的算力基础设施投资。** 泡沫的定义是价格远超价值。但如果技术迭代是指数级的，今天看起来贵的基础设施，三年后可能刚刚够用。

需要警惕的不是泡沫，而是**用线性思维做指数级世界的投资决策**。

在这个逻辑下，算电协同的投资策略应该是：

- **短期（现在-12个月）：** 抢占存量机房资源，快速交付。谁的交付速度快，谁就能先拿到客户的长期合同
- **中期（12-36个月）：** 分布式边缘节点组网，形成网络效应。节点越多，新进入者的竞争成本越高
- **长期（36个月+）：** 西部集中式基地布局，承接下一波更大规模的训练需求

## 写在最后

把分析框架从"供给侧有什么资源"倒过来，换成"用户需要什么"，很多选择就变得清晰了：

**从用户的时间维度看：**
- 急用的（这个月就要）→ 存量改造/分布式算力仓（快）
- 近期的（下季度就要）→ 边缘节点/旧改扩容（中等）
- 长期的（明后年要）→ 规划新建智算中心（战略性）

**从用户的需求维度看：**
- 要低延迟 → 边缘分布式
- 要低成本 → 西部集中式
- 要绿色合规 → 绿电+能碳管理能力决定差异化定价权
- 要供应链稳定 → 双路供电+储能备份成为标配

**新建和存量不是互斥的。客户的算力需求是持续增长的曲线——既等不起，也停不下来。** 旧改填补"等不起"的当下，新建承载"停不下来"的未来。

算电协同的投资机会，本质是**服务于"算力用户的多维需求"的能力**。谁能在正确的时间维度、以正确的交付方式、在正确的地点、以正确的成本，提供用户需要的算力——谁就能在这个赛道里赚到钱。
